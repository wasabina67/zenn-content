## はじめに

プログラミングにおいて、変数名や関数名などの命名規則は非常に重要です。
本記事では、代表的な3つの命名規則（キャメルケース、スネークケース、ケバブケース）について解説します。

## キャメルケース (camelCase)

キャメルケースは、複数の単語を連結する際に、最初の単語は小文字で始め、以降の単語の先頭を大文字にする命名規則です。
単語の区切りが大文字になることで、ラクダ（camel）のコブのように見えることから、この名前がつきました。

```
userName
firstName
getFullName
calculateTotalPrice
```

### 主な使用場面

- JavaScript、TypeScriptの変数名や関数名
- Javaのメソッド名や変数名
- JSONのキー名

### パスカルケース (PascalCase)

キャメルケースの派生として、最初の単語も大文字で始めるパスカルケース（またはアッパーキャメルケース）があります。

```
UserName
FirstName
GetFullName
CalculateTotalPrice
```

主にクラス名やコンストラクタ名に使用されます。

## スネークケース (snake_case)

スネークケースは、単語をアンダースコア（_）で区切り、すべて小文字で表記する命名規則です。
アンダースコアが蛇（snake）のように見えることから、この名前がつきました。

```
user_name
first_name
get_full_name
calculate_total_price
```

### 主な使用場面

- Pythonの変数名や関数名
- Rubyの変数名やメソッド名
- データベースのテーブル名やカラム名
- 環境変数（大文字のスネークケース：`USER_NAME`）

### コンスタントケース (CONSTANT_CASE)

スネークケースの派生として、すべて大文字で表記するコンスタントケース（またはスクリーミングスネークケース）があります。

```
MAX_VALUE
API_BASE_URL
DEFAULT_TIMEOUT
```

主に定数名に使用されます。

## ケバブケース (kebab-case)

ケバブケースは、単語をハイフン（-）で区切り、すべて小文字で表記する命名規則です。
ハイフンでつながった単語がケバブの串のように見えることから、この名前がつきました。

```
user-name
first-name
get-full-name
calculate-total-price
```

### 主な使用場面

- URLのパス（`/user-profile`）
- CSSのクラス名やID名
- HTMLの属性名（`data-user-id`）
- npmパッケージ名
- ファイル名やディレクトリ名

### 注意点

多くのプログラミング言語では、ハイフンは減算演算子として解釈されるため、変数名や関数名には使用できません。
そのため、ケバブケースは主にコード外の命名（URL、CSS、ファイル名など）で使用されます。

## まとめ

| 命名規則 | 例 | 主な用途 |
|---------|-----|---------|
| キャメルケース | `userName` | JavaScript、Javaの変数・関数名 |
| パスカルケース | `UserName` | クラス名、コンストラクタ名 |
| スネークケース | `user_name` | Python、Rubyの変数・関数名、DB |
| コンスタントケース | `USER_NAME` | 定数名 |
| ケバブケース | `user-name` | URL、CSS、ファイル名 |

プロジェクトやチームで使用する命名規則を統一することで、コードの可読性と保守性が向上します。
言語やフレームワークの慣習に従いつつ、一貫性のある命名を心がけましょう。
